# ONE LAW competition proof-loop decision

## Recommendation

Keep ONE LAW as the final Build Week concept, freeze feature scope, and use the remaining work only to complete and prove one arbitrary-law loop from ratification through tribunal.

## Decision required

Approve proof-loop completion on the current `agent/dynamic-game-loop` branch. This authorizes dynamic tribunal work, causal presentation, documentation correction, verification, deployment, playtesting, and submission packaging. It does not authorize another concept pivot or unrelated visual expansion.

## Why now

The concept has a ten-second premise and differentiated mechanic, but the deployed product does not yet prove the current dynamic implementation. More features would displace the evidence required by the judging rubric.

## Criteria

Ten-second premise, visible causal loop, dramatic climax, essential GPT-5.6 role, reliability, buildability before the deadline, differentiation, and judge-visible proof.

## Alternatives

1. **Complete the dynamic proof loop — recommended.** Best balance of idea quality, technical substance, and deadline feasibility.
2. **Submit only the cached demo.** More reliable, but weakens the arbitrary-law claim and meaningful model-use proof.
3. **Pivot or expand the concept.** Reject because it destroys accumulated implementation and proof time.

## Strongest objection

The experience could appear to be a scripted simulation using GPT-5.6 once while TypeScript predetermines the meaningful result. The response is an evidence-bound final synthesis generated from the arbitrary law and exact five rulings, while deterministic code retains authority over state, votes, and ending.

## Execution and failure policy

Pass requires a live arbitrary-law run, five rulings, evidence-cited tribunal, production deployment, adversarial review disposition, completed playtests, and an uninterrupted three-minute recording. Do not submit claims that are not visible in the deployed artifact. If live GPT-5.6 cannot be funded and verified, submit the certified demo with the limitation stated explicitly rather than narrating around it.

## Exact ask

Approve the frozen proof-loop scope above. No other scope is authorized.
